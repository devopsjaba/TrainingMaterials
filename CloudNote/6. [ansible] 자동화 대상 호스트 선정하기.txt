1. 서론
    1) 어떤 시스템의 호스트(host)를 자동화할 것인지를 선정
    2) 대상 호스트 선정이 되면 인벤토리를 통해 대상 호스트를 설정함 

2. 인벤토리를 이용한 자동화 대상 호스트 설정하기 
    1) 인벤토리 파일은 텍스트 파일임
    2) 이름=값 또는 YAML을 포함한 다양한 형식 사용하여 작성 가능  
    3) IP를 이용한 인벤토리 파일 생성 
    4) 호스트명을 이용한 인벤토리 파일 생성
        - 앤서블이 호스트명으로 접근할 수 있도록 IP를 /etc/hosts에 등록함.
          (/etc/hosts에 자동화 대상 호스트 등록)
        - 등록한 호스트명을 이용해 inventory 파일에 설정  
          (호스명을 이용한 inventory 파일 생성)  

3. 역할(Role)에 따른 호스트 그룹 설정 
    1) 호스별로 롤(역할)별 특정 작업을 수행해야 하는 경우가 발생함 
        - 여러 대로 구성할 경우 인벤토리도 유형별로 호스트를 설정 
    2) 그룹별 호스트 설정 
        - 실행 시 그룹별로 작업 처리 할 수 있음 
        - 그룹명을 대괄호([])내에 작성하고 해당 그룹에 속하는 
          호스트명이나 IP를 한줄에 하나씩 나열함
        - 호스트는 여러 그룹에 있을 수 있음.
            - 특정 호스트 집합에 앤서블을 쉽게 적용할 수 있음.                             
        - 중첩 그룹 정의 
            - 엔서블 인벤토리는 호스트 그룹에 기존에 정의된 호스트 그룹에
              포함할 수 있음.
            - 호스트 그룹 이름 생성 시 :children이라는 접미사를 추가하면 됨
    3) 범위를 사용한 호스트 사양 간소화 
        - 숫자 또는 영문자로 범위 지정할 수 있음 
        - 대괄호 사이에 시작 구문과 종료 구문을 표함함 
            - [start:end]
