1. 깃 소개 
    1) 2005년 리누스 토르발스는 깃(Git)을 세상에 소개함 
    2) 소스 코드 관리의 효율성을 위한 소프트웨어 
    3) 핵심 기능
        - 버전 관리 (Version Control) 시스템 
        - 백업 (backup)
            - 깃허브 (github)가 가장 많이 사용되고 있음
                - 온라인 저장소 or 원격 저장소 
        - 협업 (collaboration)

2. 깃 환경 설정하기
    1) 깃을 사용하기 전에 먼저 사용자 정보를 입력해야 함.
    2) 깃은 버전을 저장할 때마다 그 버전을 만든 사용자 정보도
       함께 저장함 
    3) git config --global user.name "이름"
       git config --global user.email "메일 주소"

3. 리눅스 명령
    1) 현재 디렉토리 확인하기 
        - 깃 배시 실행 후 맨 끝에 물결 표시(~)가 있음 
            - 현 위치가 홈 디렉토리(home directory)라는 의미임.
        - pwd (print working directory)
            - 현재 위치의 경로 출력    

    2) ls (list)
        - 현재 디렉토리에 어떤 파일이나 디렉토리가 있는지 확인할 때 사용 
        - -l 옵션
            - 파일이나 디렉토리의 상세 정보까지 표시함 
          -a 
            - 숨긴 파일이나 디렉토리도 함께 표시함.

    3) clear
        - 터미널 창 지우기.

    4) 디렉토리를 나타내는 기호
        - .. 
            - 현재 디렉토리의 상위 디렉토리임
        - .
            - 현재 사용자가 작업 중인 디렉토리임               
        - ~ 
            - 현재 접속 중인 사용자 디렉토리를 가리킴 

    5) 디렉토리 만들기 
        - mkdir (make directory)

    6) 디렉토리 삭제
        - rm (remove)
        - rm -r 
            - 디렉토리안에 있는 하위 디렉토리와 파일을 함께 삭제함 

    7) 터미널 종료하기 
        - exit
