1. 이벤트 (event) 
    1) 웹 브라우저나 사용자가 실행하는 어떤 동작을 말함.
    2) 사용자가 웹 문서 영역을 벗어나서 클릭하는 행위는 
       이벤트가 아님.
    3) 자바스크립트 이벤트는 주로 마우스나 키보드를 사용할 때,
       웹 문서를 불러올 때, 폼(form)에 입력할 때 발생함.

2. 문서 로딩 이벤트 
    1) 종류
        - abort (웹 문서가 완전히 로딩되기 전에 불어오기를
                멈추었을 때)
        - error (문서가 정확히 로딩되지 않았을 때) 
        - load (문서 로딩이 끝났을 때)
        - resize (문서 화면의 크기가 바뀌었을 때)
        - scroll (문서 화면이 스크롤되었을 때)
        - unload (문서를 벗어날 때)  

3. 마우스 이벤트 
    1) 종류
        - click (사용자가 HTML 요소를 클릭했을 때)
        - dblclick (사용자가 HTML 요소를 더블클릭했을 때)                             
        - mousedown (사용자가 요소에서 마우스 버튼을 눌렀을 때)
        - mouseup (요소 위에 올려놓은 마우스 버튼에서 손을 뗄 때)
        - mousemove (사용자가 요소에서 마우스 포인터를 움직일 때)
        - mouseover (마우스 포인터를 요소 위로 옮길 때)
        - mouseout (마우스 포인터가 요소를 벗어날 때)

4. 키보드 이벤트 
    1) 종류
        - keydown (키를 누르는 동안)
        - keypress (키를 눌렀을 때)
        - keyup (키에서 손을 땔 때)      
    2) 키보트 이벤트에서 키값 확인하기
        - 키값이 event객체에 담겨 있음.
            - event.code, event.key           

5. 폼(form) 이벤트 
    1) 로그인, 검색, 게시판 등 사용자가 데이터(정보)를 입력하는
       모든 요소를 가리킴 
    2) 폼 요소에 내용을 입력하면서 발생할 수 있는 다양한 이벤트.
    3) 종류
        - blur  (폼 요소에 포커스를 잃었을 때)                   
        - change (목록이나 체크 상태 등이 변경되었을 때)
            - <input>, <select>, <textarea> 태그에서 사용 
        - focus   (폼 요소에 포커스를 놓았을 때)            
        - reset   (폼이 리셋되었을 때)
        - submit  ([submit] 버튼 클릭했을 때)

6. 이벤트 처리하기 
    1) 이벤트 처리기, 이벤트 핸들러      
        - 이벤트가 발생한 HTML 요소(태그)에 직접 함수를 연결
        - 이벤트 처리기를 연결하기 위한 속성 
            - 형식 
                - <태그 on이벤트명 = 함수명>      
        - HTML 태그에 스크립트를 함께 사용함 
    
    2) 웹 요소에 함수 연결하기 
        - 형식 
            - 요소.on이벤트명 = 함수 

    3) 이벤트 리스너로 이벤트 처리           
        - 이벤트 리스너(event listener) 자주 사용함 
            - 모든 DOM 요소에서 사용 할 수 있음.
            - 모든 함수에서도 실행할 수 있음.
            - addEventListener() 메서드를 사용 
        - 형식 
            - 요소.addEventListener(이벤트, 함수, 캡춰 여부)     

7. 모달 박스(modal box) 
    1) 화면에 내용이 팝업되면서 기타 내용은 블러 처리됨 
    2) 팝업된 내용만 집중할 수 있도록 해주는 창임 

8. event 객체 
    1) 웹 문서에 발생하는 이벤트 정보를 저장하는 event 객체가 있음.
        - 이벤트가 발생하면 자동으로 이벤트와 관련된 객체가 생성됨.
    2) event 객체의 프로퍼티(property)와 메서드     
        - 메서드 
            - preventDefault()
                - 기본 동작을 취소함 
`        - 프로퍼티 
            - pageX  : 현재 문서 기준으로 이벤트 발생한 가로 위치 반환함
            - pageY  : 현재 문서 기준으로 이벤트 발생한 세로 위치 반환함 
            - screenX : 현재 화면 기준으로 이벤트 발생한 가로 위치 반환함
            - screenY : 현재 화면 기준으로 이벤트 발생한 세로 위치 반환함
            - target : 이벤트가 발생한 대상을 반환함 
            - type : 발생한 이벤트 이름 반환함      
    
    3) 마우스 이벤트에서 클릭 위치 표시하기                         

    4) 캐러셀(carousel)
        - 이미지나 콘텐츠를 슬라이드 쇼처럼 보여주는 요소 
        - 이벤트 객체 사용함 
        - 이벤트 처리 
            - 화살표 요소 가져와서 arrows로 저장함
            - click 이벤트 발생하면 왼쪽 화살표, 오른쪽 화살표 확인하기 
            - event 객체의 target 프로퍼티가 이벤트가 발생한 대상을 가리킴 

    5) 웹 문서에서 마우스 오른쪽 버튼 비활성화 시키기 

9. 이벤트 전파 
    1) 웹 요소에서 이벤트가 발생했을 때 해당 요소에서만 이벤트가 처리되는 것이
       아님.
    2) 해당 요소를 감싸고 있는 부모 요소에서도 똑같이 이벤트가 처리가 됨.                       
    3) 이벤트 전파(event propagation)라고 함.
        - 버블링, 캡처링  

    4) 이벤트 버블링
        - 브라우저에서는 기본적으로 '이벤트 버블링'이라는 전파방식을 사용함 
        - 특정 요소에서 이벤트가 발생했을 때 그 이벤트가 해당 요소뿐만 아니라 
          그 요소의 부모 요소, 부모 요소의 부모 요소에도 똑같이 발생 한것으로
          간주하는 것.  
        - 아래에서 위로 전파됨 (자식 요소에서 부모 요소로)          
        - event 객체의 이벤트 전파와 관련 있는 프로퍼티 
            - target : 이벤트가 발생한 대상                  
            - currentTarget 

    5) 이벤트 캡처링 (event capturing)
        - 웹 요소에서 이벤트가 발생하면 일단 최상위 요소에서 시작해서
          이벤트가 발행한 요소까지 차례대로 전파되는 방식임.
        - 위에서 아래로 (부모 요소에서 자식 요소로) 이벤트가 전파되는 방식임.
        - 이벤트 리스너에서 세 번째 옵션은 기본적으로 false 사용함 
            - 이벤트 캡춰링 미사용함
